MySql读写分离

MySql读写分离

1. 基本原理

  • 让主数据库处理事务性增删改操作,让从数据库执行查操作.
  • 数据库复制被用来把事务性操作导致的变更同步到从数据库

2. 为什么要读写分离

  • 数据库的增删改操作的耗时要远远大于查询操作

3. MySql支持的复制类型

  • 基于语句的复制
  • 基于行的复制，复制改变的内容
  • 混合类型的复制

4, MySql读写分离的原理

  • 读写分离就是在主服务器上修改,数据会同步到从服务器,从服务器只读,实现备份的同时也实现了数据库性能的优化
